# TI OMAP3530 — Procesador

## Arquitectura

El OMAP3530 es un System-on-Chip (SoC) de Texas Instruments basado en la arquitectura "Application Processor + DSP".

```
┌─────────────────────────────────────┐
│         OMAP3530 SoC                │
├──────────────────┬──────────────────┤
│  ARM Cortex-A8   │  DSP C64x+       │
│  @ 600MHz (720MHz│  @ 430MHz        │
│  en AR.Drone 2)  │  (fijo)          │
│  ← Linux/Kernel  │  ← DSP/BIOS      │
├──────────────────┴──────────────────┤
│  PERIFÉRICOS                        │
│  - Camera ISP (parallel + CSI)      │
│  - Display subsystem                │
│  - USB 2.0 OTG                      │
│  - MMC/SDIO                         │
│  - I2C, SPI, UART                   │
│  - Ethernet MAC                     │
│  - SGX530 3D Graphics               │
└─────────────────────────────────────┘
```

## Subsistema de Cámara (ISP)

El OMAP3530 tiene un Image Signal Processor (ISP) que:
- Soporta sensores paralelos (CAM) y seriales (CSI2)
- Procesa raw Bayer a YUV422
- Resolución máxima: 4096x4096
- Frame rate: hasta 30fps a 720p
- Escalado, recorte, balance de blancos, corrección de lente

## Subsistema de Video

- **Codec Engine**: framework que permite al ARM usar códecs en el DSP
- **DSPLink**: controla la comunicación ARM ↔ DSP
- **Resize**: escalador de imagen (hardware)
- **Color space conversion**: CSC hardware

## DSP C64x+

| Característica | Valor |
|---|---|
| Arquitectura | VLIW (Very Long Instruction Word) |
| Velocidad | 430MHz (fija) |
| ALUs | 8 (32-bit) |
| Multiplicadores | 2 (32x32→64-bit) |
| Memoria L1 | 32KB programa + 80KB datos |
| Memoria L2 | 128KB (configurable como cache/SRAM) |
| DMA | Enhanced DMA (64 canales) |

## Codec Engine

- BaseImage: `baseimage.dof` — firmware del DSP
- Códec H.264: `m4venc_sn.dll64P` — dynreg para encoder
- API: Engine_open, Engine_close, VIDENC_create, VIDENC_process
- Limitación: **solo una sesión de encoding a la vez**
