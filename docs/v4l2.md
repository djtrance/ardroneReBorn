# V4L2 — Video for Linux 2

## Descripción General

Video4Linux2 (V4L2) es la API de captura de video del kernel Linux. En el AR.Drone 2.0, se usa para acceder a las cámaras frontal e inferior.

## Dispositivos

| Device | Cámara | Resolución | Formato |
|---|---|---|---|
| `/dev/video0` | Frontal (720p) | 1280x720 @ 30fps | UYVY (YUV422) |
| `/dev/video1` | Inferior | 320x240 @ 60fps | NV12 (YUV420) |

## API Básica de Captura

### 1. Abrir dispositivo

```c
int fd = open("/dev/video0", O_RDWR);
if (fd < 0) { /* error */ }
```

### 2. Configurar formato

```c
struct v4l2_format fmt = {0};
fmt.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
fmt.fmt.pix.width = 1280;
fmt.fmt.pix.height = 720;
fmt.fmt.pix.pixelformat = V4L2_PIX_FMT_UYVY;
fmt.fmt.pix.field = V4L2_FIELD_NONE;
ioctl(fd, VIDIOC_S_FMT, &fmt);
```

### 3. Solicitar buffers

```c
struct v4l2_requestbuffers req = {0};
req.count = 4;
req.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
req.memory = V4L2_MEMORY_MMAP;
ioctl(fd, VIDIOC_REQBUFS, &req);
```

### 4. Hacer streaming

```c
enum v4l2_buf_type type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
ioctl(fd, VIDIOC_STREAMON, &type);

// En un loop:
struct v4l2_buffer buf = {0};
buf.type = V4L2_BUF_TYPE_VIDEO_CAPTURE;
buf.memory = V4L2_MEMORY_MMAP;
ioctl(fd, VIDIOC_DQBUF, &buf);  // obtener frame
// procesar buf.memoria
ioctl(fd, VIDIOC_QBUF, &buf);   // devolver buffer
```

## Integración con DSP H.264

Para enviar frames capturados al DSP:

```c
// 1. Capturar frame UYVY desde V4L2
// 2. Convertir UYVY → NV12 (necesario para el DSP)
// 3. Enviar NV12 al Codec Engine
// 4. Obtener bitstream H.264 del DSP
```

Conversión UYVY → NV12:

```c
void uyvy_to_nv12(uint8_t *uyvy, uint8_t *nv12, int width, int height) {
    int frame_size = width * height;
    uint8_t *y_plane = nv12;
    uint8_t *uv_plane = nv12 + frame_size;

    for (int i = 0; i < height; i++) {
        for (int j = 0; j < width; j += 2) {
            int uyvy_idx = (i * width + j) * 2;
            int nv12_y_idx = i * width + j;
            int nv12_uv_idx = (i / 2) * width + j;

            // Y values
            y_plane[nv12_y_idx] = uyvy[uyvy_idx + 1];      // Y0
            y_plane[nv12_y_idx + 1] = uyvy[uyvy_idx + 3];  // Y1

            // UV interleaved (cada 2 filas)
            if (i % 2 == 0) {
                uv_plane[nv12_uv_idx] = uyvy[uyvy_idx + 2];     // U
                uv_plane[nv12_uv_idx + 1] = uyvy[uyvy_idx];     // V
            }
        }
    }
}
```

## Buffer Management

El AR.Drone 2.0 tiene RAM limitada. Usar **solo 2-4 buffers** para evitar agotar memoria compartida con el DSP.
